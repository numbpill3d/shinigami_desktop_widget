[PROJECT DESCRIPTION]
───────────────────────────────────────────────────────────────
A desktop widget for Arch Linux featuring an AI-powered chat
interface with an animated 3D shinigami (death deity) companion.
Stays on desktop layer, provides periodic check-ins, and enables
conversations with various AI providers.

[TECHNOLOGY STACK]
───────────────────────────────────────────────────────────────
  • Electron - Desktop application framework
  • Three.js - 3D graphics for animated creature
  • Node.js - Backend logic and API calls
  • Pure HTML/CSS/JavaScript - No frameworks
  • electron-store - Configuration persistence

[KEY FEATURES]
───────────────────────────────────────────────────────────────
  ✓ Desktop widget (non-intrusive)
  ✓ Animated 3D shinigami creature with:
    - Floating motion
    - Pulsing eyes
    - Swaying arms
    - Orbiting ethereal wisps
    - Particle effects
  ✓ AI chat interface supporting:
    - OpenAI (GPT-4, GPT-3.5)
    - OpenRouter (multiple providers)
    - Anthropic (Claude)
  ✓ Periodic check-in questions
  ✓ Persistent chat history
  ✓ Customizable system prompts
  ✓ Export conversations
  ✓ Cyberpunk aesthetic

[QUICK START]
───────────────────────────────────────────────────────────────
1. Extract files to: ~/apps/shinigami-companion
2. Run: ./install.sh
3. Start: npm start or ./launch.sh
4. Configure: Click [CONF], add API key, click [SAVE]
5. Apply window rules: ./set-window-rules.sh

[SYSTEM REQUIREMENTS]
───────────────────────────────────────────────────────────────
  • Arch Linux or Arch-based distro
  • KDE Plasma (recommended) or other DE/WM
  • Node.js v16+
  • xdotool & wmctrl (for window management)
  • API key from OpenAI/OpenRouter/Anthropic

[CUSTOMIZATION HOTSPOTS]
───────────────────────────────────────────────────────────────
  Colors:           style.css (lines with #ff00ff, #00ff00)
  Creature:         renderer.js → createShinigami()
  Questions:        main.js → sendPeriodicQuestion()
  Check intervals:  main.js → QUESTION_INTERVALS array
  System prompt:    Settings modal in app
  API endpoints:    renderer.js → callAI functions

[WINDOW MANAGEMENT]
───────────────────────────────────────────────────────────────
KDE Plasma:   Run ./set-window-rules.sh after starting
i3wm:         Add to ~/.config/i3/config
Sway:         Add to ~/.config/sway/config
bspwm:        Add to ~/.config/bspwm/bspwmrc
Openbox:      Add to ~/.config/openbox/rc.xml

See ADVANCED.md for window manager specific configs.

[API PROVIDERS]
───────────────────────────────────────────────────────────────
OpenAI:
  • Endpoint: api.openai.com
  • Models: gpt-4, gpt-4-turbo, gpt-3.5-turbo
  • Key format: sk-...

OpenRouter:
  • Endpoint: openrouter.ai
  • Models: openai/gpt-4, anthropic/claude-3-opus, etc.
  • Key format: sk-or-...

Anthropic:
  • Endpoint: api.anthropic.com
  • Models: claude-3-opus-20240229, claude-3-sonnet-20240229
  • Key format: sk-ant-...

[PERIODIC QUESTIONS]
───────────────────────────────────────────────────────────────

Disable:      Settings → Uncheck "Enable periodic check-ins"

[TROUBLESHOOTING]
───────────────────────────────────────────────────────────────
Window issues:       Run ./set-window-rules.sh
API errors:          Check key, model name, credits
Transparent issues:  Set transparent: false in main.js
App won't start:     Check Node.js version, reinstall deps
Performance:         Reduce particles, lower FPS

See README.md and QUICKREF.txt for detailed troubleshooting.
